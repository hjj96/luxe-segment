# Подключение к серверу Reg.ru

## Проблема
Вы пытаетесь выполнить команды для Linux на Mac. Команды нужно выполнять **НА СЕРВЕРЕ**, а не на вашем компьютере.

---

## Шаг 1: Подключение к серверу

### Вариант A: Подключение по паролю

```bash
# На вашем Mac выполните:
ssh root@149.88.109.83
```

Когда спросит пароль - введите пароль от root на сервере Reg.ru.

### Вариант B: Если пароль не работает

Если получаете ошибку "Permission denied (publickey)", нужно настроить доступ:

1. **Проверьте способ доступа в панели Reg.ru:**
   - Войдите в панель управления Reg.ru
   - Найдите ваш VPS
   - Проверьте, какой способ доступа настроен (пароль или SSH ключ)

2. **Если нужен пароль:**
   - В панели Reg.ru найдите раздел "Пароли" или "Доступ"
   - Установите/сбросьте пароль для root

3. **Если нужен SSH ключ:**
   - Создайте SSH ключ на Mac:
     ```bash
     ssh-keygen -t ed25519 -C "ваш@email.com"
     # Нажмите Enter для всех вопросов
     ```
   - Скопируйте публичный ключ:
     ```bash
     cat ~/.ssh/id_ed25519.pub
     ```
   - Добавьте ключ в панели Reg.ru (раздел SSH ключи)

## Шаг 2: После подключения к серверу

**ВАЖНО:** Все следующие команды выполняйте **ПОСЛЕ** подключения к серверу!

После успешного подключения вы увидите приглашение типа:
```
root@server:~#
```

Это означает, что вы **НА СЕРВЕРЕ**. Теперь можно выполнять команды.

---

## Шаг 3: Установка ПО на сервере

После подключения к серверу выполните:

```bash
# Обновите систему
apt update && apt upgrade -y

# Установите Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Установите PM2
npm install -g pm2

# Установите Nginx и Git
apt install -y nginx git
```

## Если SSH не работает

### Проблема: "Permission denied (publickey)"

**Решение 1: Используйте пароль**
- В панели Reg.ru установите пароль для root
- Подключитесь: `ssh root@149.88.109.83`
- Введите пароль

**Решение 2: Используйте веб-консоль**
- В панели Reg.ru найдите раздел "Консоль" или "VNC"
- Откройте веб-консоль сервера
- Выполните команды через веб-интерфейс

**Решение 3: Настройте SSH ключ**
```bash
# На Mac создайте ключ
ssh-keygen -t ed25519 -C "ваш@email.com"

# Скопируйте публичный ключ
cat ~/.ssh/id_ed25519.pub

# Добавьте ключ в панели Reg.ru
```

## Важно помнить

✅ **Команды для Linux (apt, npm install -g)** выполняются **НА СЕРВЕРЕ**  
❌ **НЕ выполняйте их на Mac** - они не работают там

✅ **На Mac** вы только подключаетесь к серверу через SSH  
❌ **На Mac** не используйте `apt` - это команда для Linux

## Пошаговая инструкция

1. **На Mac:** Подключитесь к серверу
   ```bash
   ssh root@149.88.109.83
   ```

2. **На сервере:** Выполните команды установки
   ```bash
   apt update && apt upgrade -y
   curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
   apt install -y nodejs
   npm install -g pm2
   apt install -y nginx git
   ```

3. **На сервере:** Клонируйте проект
   ```bash
   mkdir -p /var/www/luxe-segment
   cd /var/www/luxe-segment
   git clone https://github.com/hjj96/luxe-segment.git .
   ```

4. **На сервере:** Настройте проект
   ```bash
   npm install --production
   nano .env  # Создайте файл с переменными
   npm run build
   pm2 start ecosystem.config.js
   ```

## Быстрая проверка

После подключения к серверу выполните:
```bash
# Проверьте, что вы на сервере
whoami
# Должно показать: root

# Проверьте операционную систему
uname -a
# Должно показать Linux, не Darwin (Mac)
```

Если видите `Darwin` - вы все еще на Mac, нужно подключиться к серверу!
