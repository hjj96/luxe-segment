# Команды для терминала - По порядку

## Что делать: Закоммитить изменения и запушить в GitHub

### ШАГ 1: Перейти в папку проекта

```bash
cd /Users/a/ЛСсайт
```

Нажмите Enter.

---

### ШАГ 2: Проверить статус

```bash
git status
```

Нажмите Enter. Покажет какие файлы изменены.

---

### ШАГ 3: Добавить все изменения

```bash
git add .
```

Нажмите Enter. Добавит все изменённые файлы.

---

### ШАГ 4: Создать коммит

```bash
git commit -m "Fix: Move code storage functions to separate file"
```

Нажмите Enter. Создаст коммит с изменениями.

---

### ШАГ 5: Отправить на GitHub

```bash
git push
```

Нажмите Enter. Отправит изменения на GitHub.

---

## Полный список команд (скопируйте и выполните по очереди):

```bash
cd /Users/a/ЛСсайт
git status
git add .
git commit -m "Fix: Move code storage functions to separate file"
git push
```

---

## Что произойдёт:

1. После `git push` изменения отправятся на GitHub
2. Vercel автоматически увидит изменения
3. Vercel запустит новую сборку
4. Через 2-3 минуты сайт будет обновлён

---

## Если что-то не работает:

### Если `git push` просит пароль:
- Используйте токен GitHub (не пароль)
- Или токен уже сохранён в Keychain - просто нажмите Enter

### Если ошибка "nothing to commit":
- Значит изменения уже закоммичены
- Просто выполните `git push`

### Если другая ошибка:
- Напишите какая ошибка - помогу исправить

---

## Готово!

После выполнения всех команд подождите 2-3 минуты и проверьте Vercel - сборка должна пройти успешно!
