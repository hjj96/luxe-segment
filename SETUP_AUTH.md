# Быстрый старт: Настройка аутентификации

## Режим разработки (без API ключей)

По умолчанию система работает в режиме разработки:
- Коды выводятся в консоль сервера
- Email коды: `[DEV] Email code for email@example.com: 123456`
- SMS коды: `[DEV] SMS code for 79991234567: 123456`

**Просто запустите сервер и тестируйте!**

## Настройка для продакшена

### 1. Создайте файл `.env` в корне проекта:

```bash
cp .env.example .env
```

### 2. Настройте JWT Secret (обязательно):

```bash
# Сгенерируйте случайный ключ
openssl rand -base64 32
```

Добавьте в `.env`:
```
JWT_SECRET=ваш-сгенерированный-ключ
```

### 3. Настройте Email (опционально):

**Resend (рекомендуется):**
1. Зарегистрируйтесь на https://resend.com
2. Создайте API ключ
3. Добавьте в `.env`:
   ```
   RESEND_API_KEY=re_xxxxxxxxxxxxx
   RESEND_FROM_EMAIL=noreply@yourdomain.com
   ```

### 4. Настройте SMS (опционально):

**SMS.ru:**
1. Зарегистрируйтесь на https://sms.ru
2. Получите API ID
3. Добавьте в `.env`:
   ```
   SMSRU_API_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```

## Как это работает

1. Пользователь вводит телефон или email
2. Система отправляет 6-значный код (SMS или email)
3. Код действителен 10 минут
4. После проверки создается сессия (JWT токен на 30 дней)
5. Токен хранится в безопасной httpOnly cookie

## SMS не приходит (сайт на Amvera / продакшен)

1. **`SMSRU_API_ID`** должен быть в переменных окружения на **этапе «Запуск»** (API отправляет SMS во время работы сайта, не при сборке).
2. Проверьте **баланс** в кабинете [sms.ru](https://sms.ru) и что **API ID** скопирован полностью (формат UUID).
3. Номер вводите в формате **+7…** или **8…** — в коде он приводится к виду `79001234567` для SMS.ru.
4. В ответе SMS.ru бывает **`status: OK` на всём ответе**, но для конкретного номера в объекте **`sms`** — ошибка (например, запрет на рассылку, неверный номер). В актуальной версии кода эта ситуация обрабатывается и на странице показывается текст ошибки.
5. Если раньше без ключа в продакшене показывалось «код отправлен», а SMS не было — задайте **`SMSRU_API_ID`** и задеплойте снова.

## Тестирование

1. Откройте http://localhost:3000/account
2. Введите телефон (например: `+79991234567` или `89991234567`) или email
3. Проверьте консоль сервера для получения кода
4. Введите код и войдите

## Безопасность

- ✅ JWT токены с httpOnly cookies
- ✅ Коды истекают через 10 минут
- ✅ Сессии действительны 30 дней
- ⚠️ В продакшене добавьте rate limiting (ограничение запросов)
- ⚠️ Замените Map на Redis/БД для хранения кодов
