# JWT_SECRET — что это и что с ним делать

## Что это

**JWT_SECRET** — это секретный ключ (случайная строка), который сервер использует, чтобы **подписывать** сессии пользователей (токены входа). Без него нельзя проверить, что «залогиненный» пользователь действительно вошёл по коду, а не подделал запрос.

- **Нигде «взять» его нельзя** — его нужно **придумать один раз** (лучше сгенерировать случайно).
- Он **не отправляется** ни в браузер, ни по сети — хранится только в `.env` на сервере.

---

## Что с ним делать

### 1. Где хранить

В файле **`.env`** в корне проекта уже есть строка:

```
JWT_SECRET=ваша-случайная-строка
```

Туда и подставляется ваш секрет. Файл `.env` в репозиторий не попадает (он в `.gitignore`).

### 2. Как сгенерировать (если будете менять)

В терминале выполните:

```bash
openssl rand -base64 32
```

Скопируйте вывод (одну длинную строку) и вставьте в `.env` как значение `JWT_SECRET=`.

### 3. На хостинге (Vercel и др.)

В настройках проекта добавьте **переменную окружения**:

- **Имя:** `JWT_SECRET`
- **Значение:** та же самая строка, что и в `.env` (можно сгенерировать новую или использовать ту же).

Без этой переменной на хостинге вход «по коду» работать не будет.

---

## Кратко

| Вопрос | Ответ |
|--------|--------|
| Где взять? | Нигде — генерируете сами: `openssl rand -base64 32` |
| Куда вписать? | В `.env` в строку `JWT_SECRET=...` |
| Зачем? | Чтобы сервер мог безопасно подписывать сессии входа |
| Нужно ли кому-то показывать? | Нет. Храните только у себя и на сервере в переменных окружения |

В вашем проекте уже подставлен сгенерированный ключ в `.env`. Менять его не обязательно, если не хотите.
