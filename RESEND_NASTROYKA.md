# Что делать в Resend

## Два варианта

### Вариант 1: Ничего не настраивать (уже работает)

Сейчас в проекте указан отправитель **`onboarding@resend.dev`** — это тестовый адрес Resend. Письма с кодами **уже уходят** на почту пользователей. Домен в Resend настраивать **не обязательно**.

---

### Вариант 2: Отправлять с вашего домена (noreply@luxesegment.ru)

Если хотите, чтобы письма шли **с вашего домена** (например `noreply@luxesegment.ru`), нужно:

1. **В Resend** — добавить домен (Domains → Add Domain → ввести `luxesegment.ru`).
2. **Скопировать DNS-записи**, которые покажет Resend (на странице «Fill in your DNS records»).
3. **В Reg.ru** (или где куплен домен) — зайти в управление DNS и **добавить эти записи**.

---

## Какие записи добавить в Reg.ru

Resend показывает три блока. Нужны минимум два.

### 1. Domain Verification (DKIM) — обязательная

| В Reg.ru укажите | Значение из Resend |
|------------------|--------------------|
| Тип | **TXT** |
| Имя/Хост | `resend._domainkey` (или `resend._domainkey.luxesegment.ru` — как подскажет Reg.ru) |
| Значение/Содержимое | Длинная строка из поля **Content** (начинается с `p=MIGfMA...`) |
| TTL | 3600 или Auto |

### 2. Enable Sending (SPF) — обязательная

**Запись MX:**

| В Reg.ru | Значение из Resend |
|----------|--------------------|
| Тип | **MX** |
| Имя | `send` |
| Значение | То, что в **Content** у MX (например `feedback-smtp...amazonses.com`) |
| Приоритет | 10 |

**Запись TXT:**

| В Reg.ru | Значение из Resend |
|----------|--------------------|
| Тип | **TXT** |
| Имя | `send` |
| Значение | Строка из **Content** (начинается с `v=spf1...`) |

### 3. DMARC (Optional) — по желанию

| В Reg.ru | Значение из Resend |
|----------|--------------------|
| Тип | **TXT** |
| Имя | `_dmarc` |
| Значение | `v=DMARC1; p=none;` |

---

## После добавления записей

1. Подождать 5–30 минут (обновление DNS).
2. В Resend нажать кнопку **«I've added the records»**.
3. Дождаться, пока Resend проверит записи (статус домена станет Verified).
4. В проекте в **`.env`** и в **Vercel → Environment Variables** поменять:
   ```env
   RESEND_FROM_EMAIL=noreply@luxesegment.ru
   ```
5. Сделать Redeploy на Vercel.

После этого письма будут уходить с адреса `noreply@luxesegment.ru`.

## Кратко

| Цель | Действие |
|------|-----------|
| Письма уже приходят с onboarding@resend.dev | Ничего не делать в Resend. |
| Хочу письма с noreply@luxesegment.ru | В Resend добавить домен → скопировать DNS → добавить эти записи в Reg.ru → нажать «I've added the records» → поменять RESEND_FROM_EMAIL и редеплой. |
