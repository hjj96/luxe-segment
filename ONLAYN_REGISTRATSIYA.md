# Онлайн-регистрация и вход

На сайте уже есть **вход без пароля**: пользователь вводит телефон или email, получает код и входит. Отдельной «регистрации» нет — первый вход по коду и есть «регистрация».

---

## Где это находится

- **Страница входа:** на сайте пункт меню **«Аккаунт»** или адрес **`/account`**.
- Пользователь вводит **телефон** или **email** → нажимает «Получить код» → вводит **код** → попадает в личный кабинет.

---

## Как это работает (для пользователя)

1. Зайти на страницу **Аккаунт** (или `/account`).
2. Ввести **номер телефона** (например `+79991234567` или `89991234567`) или **email**.
3. Нажать **«Получить код»**.
4. Получить код в SMS или на почту (в зависимости от того, что ввёл).
5. Ввести **код** и нажать **«Войти»**.
6. После этого пользователь «зарегистрирован» и авторизован — открывается личный кабинет (избранное, корзина, выход).

Пароль не нужен: вход только по коду.

---

## Как протестировать у себя (без SMS и email)

1. Запустите сайт: `npm run dev`.
2. Откройте в браузере: **http://localhost:3000/account**.
3. Введите любой телефон (например `79991234567`) или email (например `test@test.ru`).
4. Нажмите «Получить код».
5. **Код появится в терминале**, где запущен `npm run dev` (сообщение вида `[DEV] Email code for ... : 123456` или SMS).
6. Введите этот код на странице и нажмите «Войти».

Так можно проверять вход без настройки SMS и email.

## Как включить настоящую отправку кодов (для боевого сайта)

Чтобы коды реально приходили на телефон или на почту, нужно добавить ключи в переменные окружения.

### 1. Файл с настройками

В корне проекта создайте файл **`.env`** (если его ещё нет), скопировав пример:

```bash
cp .env.example .env
```

В **`.env`** задайте минимум:

```env
# Обязательно: свой секретный ключ (сгенерируйте один раз)
JWT_SECRET=ваш-длинный-случайный-ключ

# Для продакшена
NODE_ENV=production
```

Сгенерировать ключ можно так (в терминале):

```bash
openssl rand -base64 32
```

Скопируйте вывод и вставьте в `.env` как значение `JWT_SECRET`.

### 2. Отправка кода на **email**

1. Зарегистрируйтесь на **https://resend.com**.
2. Создайте API-ключ.
3. В **`.env`** добавьте:

```env
RESEND_API_KEY=re_xxxxxxxxxxxxx
RESEND_FROM_EMAIL=noreply@ваш-домен.com
```

После этого коды на email будут уходить через Resend.

### 3. Отправка кода по **SMS**

1. Зарегистрируйтесь на **https://sms.ru**.
2. В личном кабинете получите **API ID**.
3. В **`.env`** добавьте:

```env
SMSRU_API_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

После этого коды на телефон будут уходить через SMS.ru (по тарифам сервиса).

## На Vercel (или другом хостинге)

В настройках проекта добавьте **Environment Variables** (переменные окружения) с теми же именами:

- `JWT_SECRET`
- `RESEND_API_KEY` и `RESEND_FROM_EMAIL` (если нужна отправка на email)
- `SMSRU_API_ID` (если нужна отправка SMS)
- `NODE_ENV=production`

После сохранения переменных перезапустите деплой (Redeploy).
