# Быстрый деплой на Vercel (5 минут)

## Самый простой способ

### Вариант 1: Через веб-интерфейс (рекомендуется)

1. **Закоммитьте код в Git:**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```

2. **Загрузите на GitHub:**
   - Создайте репозиторий на GitHub
   - Загрузите код:
     ```bash
     git remote add origin https://github.com/ваш-username/ваш-репозиторий.git
     git push -u origin main
     ```

3. **Деплой на Vercel:**
   - Откройте https://vercel.com
   - Нажмите "Sign Up" → войдите через GitHub
   - Нажмите "Add New Project"
   - Выберите ваш репозиторий
   - Нажмите "Deploy"

4. **Настройте переменные окружения:**
   - После деплоя: Settings → Environment Variables
   - Добавьте:
     ```
     JWT_SECRET=сгенерируйте-ключ-через-openssl-rand-base64-32
     NODE_ENV=production
     ```
   - Если используете Email/SMS, добавьте соответствующие ключи

5. **Перезапустите деплой:**
   - Deployments → три точки → Redeploy

Готово! Ваш сайт доступен по адресу `ваш-проект.vercel.app`

---

### Вариант 2: Через CLI

```bash
# 1. Установите Vercel CLI
npm i -g vercel

# 2. Войдите
vercel login

# 3. Деплой
vercel

# 4. Продакшен деплой
vercel --prod
```

---

## Что нужно настроить

### Обязательно:
- ✅ `JWT_SECRET` — сгенерируйте случайный ключ
- ✅ `NODE_ENV=production`

### Опционально (если используете):
- `RESEND_API_KEY` — для отправки email
- `SMSRU_API_ID` — для отправки SMS
- `RESEND_FROM_EMAIL` — email отправителя

---

## Проверка после деплоя

1. Откройте ваш сайт
2. Проверьте все страницы
3. Протестируйте авторизацию:
   - Войдите в аккаунт
   - Проверьте, что код приходит (в консоли сервера или на email/SMS)

---

## Подключение своего домена

1. В Vercel: Settings → Domains
2. Добавьте ваш домен
3. Следуйте инструкциям по настройке DNS

---

## Проблемы?

- **Ошибка сборки:** Проверьте `npm run build` локально
- **Переменные не работают:** Убедитесь, что они добавлены и перезапустите деплой
- **API не работает:** Проверьте, что используете правильный URL (не localhost)

Подробная инструкция: см. `DEPLOY.md`
